《客户关系管理》笔记

Created2022-02-06|Updated2023-05-06|Books
|Word count:7.4k|Reading time:21min|Post View:

基本概念

定义和内涵

客户关系管理的理论基础来源于西方的市场营销理论。随着企业战略中心从“以产品为核心”向”以客户为中心“转变，客户已经成为企业最重要的资源。客户关系管理的出现把对关系的管理从营销领域扩大到整个企业范围，从一个智能的战略演变成了企业的整体战略。

  1. 客户关系管理是现代管理理念、管理模式和信息技术三位一体的整合，其核心思想是以客户为中心，将企业的客户视为最重要的企业资产。
  2. 客户关系管理的目的是实现顾客价值最大化和企业收益最大化之间的平衡，即客户与企业的双赢；
  3. 客户关系管理是企业战略的一种，需要企业全方位的资源整合和支持；
  4. 客户关系管理离不开信息技术的支持；

功能

  1. 互动管理
  2. 营运管理
    1. 营销管理
    2. 销售管理
    3. 客户服务与支持
  3. 商务智能（BI）
  4. 系统整合（ERP、SCM）

潜在收益

  1. 建立客户忠诚；
  2. 增加长期收益；
  3. 助推企业实现“以客户为中心”的管理策略；
  4. 提升企业的核心竞争力
    1. 提高企业整体营销、销售和服务活动的有效性的同时，提高企业对其业务活动的管理水平；
    2. 对打造企业核心竞争力的战略决策能力和总体规划能力起到保障和促进作用；
    3. 通过技术上的先进性来保证企业核心竞争力的持续提高

潜在成本

  1. 信息基础设施的投入；
  2. 处理变更的成本

理论基础

顾客价值

  1. 定义: 顾客对那些产品的属性、属性表现以及从使用中引起的有利于或阻止顾客在使用状态下取得他们的目的和目标的结果的偏好及评估（核心价值+附加价值）
  2. 特征
    1. 主观性
    2. 情景依赖性
    3. 层次性
    4. 动态性
    5. 相对性
  3. 驱动因素：企业通过价值活动对企业进行顾客价值创造和传递产生影响的因素
    1. 技术：产品创新、服务、物流和技术支持
    2. 品牌：品牌意识、品牌认同和品牌忠诚
    3. 产品：价格、质量和便利性
    4. 知识：客户和企业相互促进消费的经验积累
    5. 关系：信任、情感氛围、情感联络和转移成本。

顾客满意

  1. 定义：指一个人通过对一种产品的可感知效果与他的期望值比较后，所形成的愉悦或失望的感觉状态
  2. 抽象：
    1. 是主观的情感反应
    2. 基于期望的相对结果
    3. 具有个体性
  3. 具体衡量
    1. 投诉与建议系统
    2. 顾客满意度调查：随机抽样
    3. 佯装的购物者：神秘顾客
    4. 分析流失的顾客

顾客忠诚

  1. 定义：顾客在对产品或服务的满意度不断提高的基础上，重复购买该产品或服务，以及向他人热情推荐该产品或服务的行为表现。
  2. 特征：
    1. 顾客对企业有明显感情倾向；
    2. 顾客在购买行为上有实际的重复反应；
    3. 顾客在长期内有偏好；
    4. 顾客对新产品或服务几乎没有顾虑地首先购买；
    5. 顾客能承受企业有限的提价，并能够抵制竞争者的降价或倾销；
  3. 意义：
    1. 更多的利润；
    2. 可以带来新的顾客；
    3. 提供意见和建议，重提为企业改进和提高管理水平、服务质量、设计和开发新产品或服务提供有益的参考；
    4. 更加容易处理不满意顾客的投诉和抱怨；
    5. 忠诚顾客群体构成企业坚实的市场基础
  4. 衡量：
    1. 复购率；
    2. 产品或服务购买的种类、数量与购买的百分比；
    3. 购买时间；
    4. 价格敏感度；
    5. 对竞品的态度；
    6. 对产品质量事故的承受能力；
  5. 顾客价值、顾客满意与顾客忠诚的关系

关系营销

概念

  1. 实质：致力于与顾客建立和维持一种良好的关系，以谋得长期的利益；
  2. 以系统论为基本思想，核心是企业建立和发展与顾客、供应商、分销商、政府机构及其他公众的良好关系

特征

  1. 信息沟通的双向性；
  2. 战略过程的协同性；
  3. 营销活动的互利性；
  4. 信息反馈的及时性；
  5. 与交易营销的区别

层次

  1. 一级关系营销：频率营销或保持性营销(短期联系)；
  2. 二级关系营销：社会性联系（在财务性关系基础上寻求与顾客建立社会性联系）；
  3. 三级关系营销：关系营销（结构性、系统性的联系）

类型

  1. 基本关系营销；
  2. 被动式关系营销；
  3. 负责式关系营销；
  4. 主动式关系营销；
  5. 伙伴式关系营销；

价值

  1. 关系是减少不确定性的手段；
  2. 促进销售、降低交易成本；
  3. 利用外部资源实现效率；
  4. 可使企业获得经济价值之外的社会价值；

数据库营销

概念

  1. 产生（建立在直复营销和关系营销基础之上）
  2. 发展历程
  3. 定义：使用营销数据，建立统计模型，通过对客户个体层次信息数据深入观察分析从而展开有效的营销；

优势

  1. 帮助企业准确找到目标顾客；
  2. 有利于发现新的市场机会和提供新产品、新服务；
  3. 有助于于顾客建立长期、良好的关系

CRM系统

概述

一般模型

工作目标

  1. 提高流程的自动化水平；
  2. 提高企业的数据分析水平

功能模块

  1. 客户接触中心子系统（协作性CRM）
  2. 业务操作自动化子系统（操作性CRM）
  3. 数据分析管理子系统（分析型CRM）

特征

  1. 综合性
  2. 集成性
  3. 智能化
  4. 高新技术特征

业务功能模块

销售自动化（SFA）

  1. SFA的功能结构：销售机会管理、活动管理、客户管理、联络人管理等；
  2. 销售机会管理是销售自动化最基本的功能和核心部件，整个SFA围绕销售机会展开；
  3. 销售自动化的主要功能包括
    1. 销售机会管理（销售机会的挖掘、确认、谈判、机会报价、结束销售机会、机会分析）；
    2. 订单管理（CRM和ERP集成的桥梁）；
    3. 合同管理；
    4. 活动管理；
    5. 客户管理（CRM的基础）；
    6. 产品管理（基础模块）；
      1. 产品信息；
      2. 产品构造；
      3. 产品定价；
      4. 产品交易记录。
    7. 销售预测和分析工具；

营销自动化（MA）

  1. 功能结构：营销方案管理、市场预算管理、宣传资料管理、促销管理、媒体管理、渠道管理、营销方案执行管理、营销方案衡量指标管理
  2. 主要功能：
    1. 利用已有营销信息和知识，控制市场营销方案循环，使得营销投资回报最大化；
    2. 增强市场营销部门通过多个渠道进行多个市场营销活动的能力，并优化营销流程；
    3. 实现与客户一对一个性化沟通，从而提高产品或服务渗透率以及客户忠诚度

客户服务自动化（CSSA）

  1. 功能结构
  2. 主要功能
    1. 服务条款管理
    2. 服务要求管理
    3. 服务订单管理
    4. 产品维修记录
    5. 问题解决方案
    6. 智能化范文管理
    7. 客户投诉管理

发展与融合

企业资源计划(ERP)

  1. 概念：
    1. 企业资源计划（ERP）是建立在IT技术之上的，以系统化的管理思想为企业决策层及员工提供决策运行手段的管理平台；
    2. 主要宗旨是对企业所拥有的人财物信息时间空间等资源进行综合平衡和优化管理，协调各部门围绕市场导向开展业务活动、提高核心竞争力；
  2. 历程：
    1. 物料需求计划阶段（MRP）；
    2. 闭环MRP生产计划与控制系统阶段；
    3. 制造资源计划阶段（MRPII）；
    4. 企业资源计划阶段（ERP）
  3. 功能模块：
    1. 生产控制管理模块；
    2. 物流管理模块；
    3. 财务管理模块；
    4. 人力资源管理模块；
  4. ERP与CRM的整合：
    1. 客户挖掘；
    2. 客户管理；
    3. 产品管理；
    4. 营销管理；
    5. 工作流管理；
    6. 决策支持；

供应链与供应链管理（SCM）

  1. 概念：
    1. 供应链是围绕核心企业，通过对信息流、物流、资金流的控制，从原材料、制成中间产品到最终产品，最后由销售网络把产品送到客户手中，将供货商、制造者、零售商、最终客户连成一个整体的功能性网链结构模式；
    2. 供应链管理（SCM）是通过前馈的信息流和反馈的物料流以及信息流，将供应商、制造商、分销商、零售商直到最终用户连接成一个整体的现代管理思想和方法，是企业信息化建设的高级阶段；
  2. 内容（注重总的物流成本与用户服务水平的关系）：
    1. 供应（Supply）
    2. 生产计划（Schedule Plan）
    3. 物流（Logistics）
    4. 需求（Demands）
  3. 整合范围：
    1. 客户数据管理；
    2. 伙伴关系管理；
    3. 销售管理；
    4. 营销管理；
    5. 客户服务；
    6. 工作流管理

客户分析

识别客户信息

客户识别的基本概念

  1. 客户的界定
    1. 客户是企业提供产品和服务的对象，客户不仅包括最终消费者，还包括分销商；
    2. 客户可以划分为最终客户和中间客户；
  2. 客户识别的含义：通过技术手段 ，根据相关数据，事先确定对企业有意义的客户作为客户关系管理的实施对象。
    1. 识别客户的目的不是了解那些客户是我们所需的，而在于识别每一个客户并将不同的数据、特征相连接，构成对客户的完整印象。
  3. 客户识别活动
    1. 定义识别标志
    2. 客户信息收集
    3. 客户信息的连接与整合
    4. 客户信息存储与更新
    5. 客户信息分析
    6. 保证客户信息安全

识别客户的信息类型

  1. 个人客户的信息类型
    1. 基本信息
    2. 心理信息
    3. 行为信息
  2. 组织客户的信息类型
    1. 基本信息
    2. 业务状况
    3. 交易状况
    4. 主要负责人信息

收集客户信息的途径和方法

  1. 途径：
    1. 直接渠道；
    2. 间接渠道；
  2. 方法：
    1. 人员访谈法；
    2. 观察法；
    3. 调查问卷法；
    4. 其他方法；

理解客户差异

  1. 客户差异最重要的方面是客户需求和客户价值；
  2. 企业应当在客户价值分析的基础之上进行细致的分类，进而最大程度满足不同用户的需求

客户需求分析

概述

  1. 定义：需求=购买能力+购买欲望
  2. 特征：
    1. 客户需求可以根据其自然属性来定位
    2. 客户需求是动态变化的
    3. 客户需求与客户价值经常会相互关联
    4. 最基本的人类需求是心理需求

细分

客户需求细分是按照客户的欲望与需求把一个总体市场划分成若干个具有共同特征的子市场的过程

  1. 地理变量
  2. 人文变量
  3. 心理变量
  4. 行为变量

客户价值分析

客户价值的界定

客户价值不单是客户直接购买而为企业带来的利润贡献，更是客户在其整个生命周期内为企业创造的所有价值的总和。

  1. 客户价值的构成：口碑价值、价格优势、学习和创新价值、战略价值
  2. 客户价值的大小——客户终身价值

客户价值的衡量和分类

  1. 客户终身价值分析法
  2. RFM模型（R-最近购买时间、F-购买评率、M-总购买金额）

客户关系建立

大规模定制建立客户关系

  1. 大规模定制的产生和发展：
    1. 大规模生产方式 MassProduction
    2. 柔性制造系统 Flexible Manufacture System, FMS
    3. 计算机集成系统 Computer Intergrated Manufacturing Systems， CIMS
    4. 大规模定制生产 Mass Customization
  2. 定义：
    1. 以客户需求为导向，以先进的制造技术、信息技术和管理技术为手段
    2. 通过大规模生产的组织形式满足单个或者小批量顾客定制产品的需求，
    3. 以此展开相应的企业活动
  3. 特点：先进技术是基础和保障、快速反应和低成本是支柱、个性化是核心
    1. 以客户需求为导向；
    2. 以先进的信息技术与制造技术为依托；
    3. 以质量为前提；
    4. 以敏捷为标志；
    5. 以供应链管理为手段；
  4. 类型：
    1. 完全化定制
    2. 设计定制
    3. 制造定制
    4. 装配定制
    5. 销售定制
    6. 服务定制
    7. 标准化定制
  5. 效益：
    1. 客户效益
      1. 满足客户个性化的产品需求
      2. 满足客户多层次的需求
      3. 信任
    2. 企业效益
      1. 提高客户的忠诚度
      2. 降低成本
      3. 提高外部资源的利用水平和效率

条件和要求

条件

  1. 生产者必须具有迅速获取消费者定制需求的能力，这是前提；
  2. 企业的产品可大规模定制化，这是基本条件；
  3. 企业具有敏捷的产品开发和柔性制造技术，这是支撑，是核心能力和资源；
  4. 供应链适合大规模定制；
  5. 企业具有与大规模定制生产方式匹配的组织系统，这是决定条件；

要求

  1. 能够全面、系统地获取和管理客户的个性化需求
  2. 能够对客户信息进行系统性的描述和分析
  3. 具备较强的集成性
  4. 能够构建企业与客户间的“学习型关系”

互联网+背景下的大规模定制

  1. 提高客户参与行为
  2. 帮助企业突破与客户的沟通渠道障碍
  3. 依托虚拟市场的规模经济与客户融合效应

营销策略

基于产品整体概念的大规模定制拓展

  1. 产品整体概念
    1. 核心利益
    2. 基础产品
    3. 期望产品
    4. 附加产品
    5. 潜在产品
  2. 大规模定制产品的层次
  3. 柔性菜单定价和主流化锁定

构建联盟式营销渠道

  1. 联盟营销网络的界定
    1. 营销导向型企业以多样化的半结合方式与供应商、最终消费者、竞争者和互补者建立的企业联盟
    2. 联盟营销网络以企业间资源共享为主要特征，充分发挥各个企业的核心优势；
  2. 纵横向营销联盟渠道的设计与建立
    1. 纵向：
      1. 有创造新价值的潜能
      2. 有共同的价值观
      3. 对伙伴关系的态度
      4. 与供应商的目标一致
    2. 横向：
      1. 有互补关系
      2. 有相同的目标市场
      3. 存在季节性需求关系
      4. 有互补的业务流程

一对一沟通

客户保持和客户流失

客户保持概述

内涵

客户保持是企业维持已建立的客户关系，使客户不断复购其产品和服务的过程

  1. 客户重复购买意向反映客户是否愿意保持与企业的关系；
  2. 客户长期关系意向是客户期望与企业保持长期关系的一种倾向；
  3. 客户保持率是用来评估企业保持客户效果的指标
  4. 客户保持和客户忠诚是密切相关的概念

意义

  1. 客户保持比客户吸引更有利于企业成本的降低；
  2. 忠诚客户提升企业的口碑效应
  3. 客户流失阻碍企业市场的扩展

影响客户保持的因素

  1. 客户满意
  2. 顾客价值
  3. 转换成本

CRM系统中的客户关系保持工作

  1. 利用分析型CRM对用户数据进行分析处理

客户流失分析

客户流失概述

  1. 定义：客户终止购买企业的产品或服务的行为
    1. 客户从本企业转到同行业的竞争对手；
    2. 客户平均消费量降低，从高价值客户转变为低价值客户；
    3. 客户专享购买和使用本企业的其他品牌或同一品牌的其他档次产品或服务
  2. 流失原因：
    1. 企业自身原因：
      1. 主动放弃客户；
      2. 企业过失;
      3. 人员流动。
    2. 竞争对手原因：
      1. 价格优惠
      2. 具有产品或服务优势
    3. 客户自身原因：
      1. 客观因素变化
      2. 尝试心里
      3. 趋众心理
      4. 购买决策人员变化；
      5. 宏观营销环境变化的原因
  3. 客户流失的识别与预测
    1. 识别指标：
      1. 客户指标
      2. 市场指标
      3. 收入利润指标
      4. 竞争力指标
    2. 客户流失预测

客户流失管理

  1. 过程：
    1. 预警阶段
    2. 分析阶段
    3. 实施阶段
    4. 评估阶段
  2. 决策误区：
    1. 盲目争取挽留所有流失客户
    2. 不择手段
    3. 事后应对

正确处理客户投诉

  1. 树立对客户投诉的正确认识
    1. 阻止客户流失
    2. 提高企业美誉度

  2. 分析不满意客户不投诉的原因

客户互动

基本概念

客户互动的内涵

  1. 客户互动是双向的信息沟通过程；
  2. 客户互动双方都可以控制互动过程；
  3. 客户互动是一个持续的动态过程；
  4. 客户互动的双重性拓展；

客户互动管理的概念模型

客户互动管理是企业整合多种互动渠道，通过及时准确的客户服务，积累客户信息，实现高效、直接、系统往复的沟通过程。

通常包含以下4个要素：

  1. 数据库支持；
  2. 整合的多渠道沟通系统；
  3. 员工促进；
  4. 及时准确的服务；

客户互动的类型

  2. 人际互动

基于服务质量的客户互动提升

服务质量的概述

  1. 定义：顾客所感知的质量
    1. 服务质量难以评估
    2. 服务质量的感知来自顾客期望与实际服务的比较结果
    3. 对服务质量的评估不仅包括对服务结果的评估，还包含对服务过程的评估
  2. 构成：
    1. 技术性质量：服务过程的产出
    2. 功能性质量：消费者是如何去的服务的结果的过程
  3. 质量差距模型：
    1. 顾客对服务的期望与公司管理层对顾客期望的认知之间的差距
    2. 管理层对顾客期望的认知和服务质量规范之间的差距
    3. 服务质量规范与服务传递之间的差距
    4. 服务传递过程与针对消费者的外部沟通之间的差距

客户数据仓库

数据仓库概述

基本概念

  1. 定义：面向主题、集成的及随时间变化的，但信息本身又相对稳定的数据集合
  2. 特点：
    1. 面向主题
    2. 集成性
    3. 数据随时间变化
    4. 相对稳定性
  3. 数据仓库的数据组织：早期细节级、当前细节级、轻度综合数据级和高度综合数据级
    1. 重要概念：
      1. 元数据
      2. 粒度
      3. 数据分割

数据仓库的体系结构

  1. 数据源：各种关系型数据库、桌面数据么、文本文件、web数据；
  2. ETL工具：数据抽取、转换和加载，其功能是对数据源进行清理、转换，对数据进行重新组织和加工，再装载到数据仓库；
  3. 数据仓库：系统的中心，存储数据，分为企业级和部门级，通常成为数据集市；
  4. OLAP服务器：基于数据仓库的信息分析处理过程，是数据仓库的用户接口部分，为分析查询提供快速响应；
  5. 前端工具：面向用户的一整套数据访问和分析工具，提供数据访问和深入分析功能；
  6. 元数据：数据定义和管理作用，是关于数据的数据
    1. 分为管理元数据和用户元数据
    2. 内容包括表的结构、表的属性、数据的属性、数据源到数据仓库的映射、数据模型的规格说明、日志和公用例行程序

数据库与数据仓库

数据仓库中的OLAP

OLAP的定义

联机分析处理是使分析人员能够迅速、一致、交互地从各个方面观察信息，这些信息是从原始数据直接转换过来，它们以用户容易理解的方式反应企业的真实情况。

OLAP的特点

  1. 快速
  2. 可分析性
  3. 多维性
  4. 信息性
  5. 共享性

结构分类

  1. MOLAP：基于多维数据库的多维OLAP
    1. 使用专有的多维数据库；
    2. 迅速相应决策分析人员的请求；
    3. 使用二维表存储；
    4. 使用预处理的操作，灵活性欠佳；
  2. ROLAP：基于关系型数据库的关系OLAP
    1. 使用关系型数据库；
    2. 管理和维护复杂；
    3. 灵活性强
  3. 选择：
    1. 查询性能；
    2. 对维数动态变化的适应能力；
    3. 对数据变化的适应能力
    4. 对大数据量的适应能力
    5. 分析查询能力

多维数据模型与OLAP

  1. 基本概念：
    1. 变量：数据度量指标
    2. 维：观察数据的特定角度
    3. 维的层次： 观察数据的角度还可以存在细节程度不同的多个描述方面
    4. 维成员： 维的一个取值
    5. 多维数组： 多个变量和维组成的数据表示结构
    6. 数据单元：多维数组的取值
  2. 多维数据的结构模型
    1. 事实表
    2. 维表
    3. 星型模型
    4. 雪花模型
  3. 多维数据模型上的OLAP操作
    1. 钻取
      1. 下钻
      2. 上卷
    2. 切片
    3. 切块
    4. 旋转
